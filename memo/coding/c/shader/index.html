GLSLのバージョン

oF上で動くのは120と150。全然言語の仕様が違うっぽい。

fragmentシェーダーでgl_FragColorに入れるのは120

150はOpenGL3.2以降でサポート。 150ではシェーダー入出力の宣言はin,outを使う。attribute,varyingは使わない。

バージョンは.frag,.vertファイルの最初に

#version 120

ってかく。

oFでは、デフォルトではOpenGL2.1が使われる。 ただし、main.cppにシェーダーを使う設定

#include "ofGLProgrammableRenderer.h"
int main(){
  ofSetCurrentRenderer(ofGLProgrammableRenderer::TYPE);
}

とすると、OpenGL4.1、GLSL4.1が使われるようになる。 その場合、varyingなどがエラーになってしまうので要注意。

ofxPostGlitchはGL2系で書かれている。サンプルがまだまだ多いのは2系

テクスチャ

oFからテクスチャをシェーダーに渡す

shader.setUniformTexture("texture", img, 0);

フラグメントシェーダからアクセス

uniform sampler2DRect texture;　// 型に注意。sampler2Dではダメ。
varying vec2 vTexCoord;
void main() {
// texture2DRect()関数で色情報にアクセス
// vTexCoordは、描画する四角形のテクスチャのuv座標がわたれば良い。
// 頂点シェーダ側で、vTexCoord =  gl_Vertex.xy; とするのが無難。
// （フラグメントシェーダ側ではvarying変数として線形補間された値が入る）
vec4 samplerColor = texture2DRect(texture, vTexCoord);
gl_FragColor = samplerColor
}
