20150115vagrant開発環境構築

20150115 vagrant開発環境構築

現状

  • php-fpm,nginxはサービスに登録してある
  • postgresはサービスに登録されていない（/etc/rc.confにはpostgresql_enable=“YES”は書いてある）
  • 192.168.33.10にアクセスすると403 directory index of “/www/docs/d00.tampen.jp/” is forbidden
  • 192.168.33.10/info.phpは見られる
  • 192.168.33.10/index.phpは PHP message: PHP Fatal error: Call to undefined function mb_language() in /www/docs/d00.tampen.jp/protected/controller/CoreController.php on line 28” でエラー

解決するべきこと

  • postgresの起動 done
  • nginxでのindexの設定 done
  • phpファイルでのパス関係の設定 done

Postgresの起動

参考：http://symfoware.blog68.fc2.com/blog-entry-1219.html 初期化 /usr/local/etc/rc.d/postgresql initdb エラー：/usr/local/pgsql/のユーザー，グループともにpgsqlであるため． /usr/local/pgsql/dataはpostgresユーザーになっている． 一旦/usr/local/pgsql/dataを削除，再度

/usr/local/etc/rc.d/postgresql initdb

初期化OK pgsqlユーザーで初期化された

/usr/local/etc/rc.d/postgresql start
sudo su - pgsql
psql postgres # 管理データベース"postgres"を指定して接続

データベースの作成・テーブルのリストア

createdb -E UTF8 tampen_dev
psql tampen_dev < tampen_schema.sql

エラー． 下記を解決する必要がある．

  • role:postgresの作成
  • pg_bigmのインストール

pg_bigmのインストール

curl,またはwgetでソースコードのダウンロード 注意点：URLに?や&が入る場合は”でURLをかこって区切らないようにしてあげる必要がある

$ curl -L -o curl.tar.gz 'http://sourceforge.jp/frs/redir.php?m=osdn&f=%2Fpgbigm%2F59914%2Fpg_bigm-1.1-20131122.tar.gz'
$ wget -O tat.tar.gz 'http://sourceforge.jp/frs/redir.php?m=osdn&f=%2Fpgbigm%2F59914%2Fpg_bigm-1.1-20131122.tar.gz'

インストール

$ gmake USE_PGXS=1 PG_CONFIG=/usr/local/bin/pg_config
$ sudo gmake USE_PGXS=1 PG_CONFIG=/usr/local/bin/pg_config install

再度postgresの初期化

# /usr/local/etc/rc.d/postgresql initdb
# /usr/local/etc/rc.d/postgresql start
# sudo su - pgsql
$ createdb -E UTF8 tampen_test

role:postgresの作成

CREATE ROLE postgres WITH SUPERUSER;

リストア

phpファイルでのパス関係の設定

http://php1st.com/419/

デフォルトでは，mb_ではじまるマルチバイト関数が無効になっている →php.iniファイルを編集

/usr/local/lib/php/20100525/php_mbstring.dllが存在しない

pkgの場合は別途mbstringのpkgをインストールしないといけない． php5-mbstring-5.4.36 をインストール

/usr/local/lib/php/20100525/mbstring.so ができたけど，php_mbstring.dllはできない． dll:windowds so:LINUX php.iniのphp_mbstring.dllをmbstring.soにする． →Module ‘mbstring’ already loaded in Unknown on line 0 コメントアウトをもとに戻す pkgでインストールすると自動的に読み込む設定になっているらしい．

session_start()が見つからない

pkg install php5-session-5.4.36 でphp-fpm再起動でかいけつ

MDB2がない

MDB2:データベースの違いに関係なくSQLを扱うためのAPI．PEARで管理されている PEAR:PHPのライブラリ・パッケージ管理システム

PEARはPHPインストール時に一緒にインストールされている．

pear install MDB2
pkg install php5-pgsql-5.4.36
pear install pear/MDB2#pgsql

データベースに接続できない

http://www.phpbook.jp/pear/pear_db/index2.html PHPのDSNの書式 phptype(dbsyntax)://username:password@protocol+hostspec/database?option=value

tampen_testじゃなくてtampen_devを作ってしまっていた． tampen_test作成

postgresにログインオプションをつけていなかった ALTER ROLE postgres WITH LOGIN

Call to undefined function gzcompress() http://lists.horde.org/archives/horde/Week-of-Mon-20080225/034810.html pkg install php5-zlib-5.4.36

nginxでのindexの設定

indexにindex.phpが含まれていなかったのが原因． →追加したらOK

新しいvagrantでpingが通らない

参考：http://labs.septeni.co.jp/?p=966

プライベートネットワークを設定する必要がある． 具体的には，ゲストOSでifconfigコマンドでネットワーク環境の確認をする． ここに192.168.33.10などが入っているのかどうかをまず確認する．

ないなら，Vagrantfileが何らかの理由で反映されていないことが分かる． Vagrantfileが反映されていない理由は？

vagrant haltでシャットダウンできない

Vagrantfileに

config.ssh.shell = 'sh'

を追加すればOK https://github.com/opscode/vagrant-omnibus/issues/69

FreeBSDはデフォルトのシェルがsh Vagrantが操作する際に使うシェルはbash -l 自分でpkg install bashをした後なら問題なく動くけど，まだ入れていないと， vagrantによってFreeBSDを操作することができない．

https://docs.vagrantup.com/v2/vagrantfile/ssh_settings.html
