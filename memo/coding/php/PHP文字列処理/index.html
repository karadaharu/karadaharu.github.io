PHP文字列処理

PHP文字列処理

クォーテーションの違い

ダブルクオーテーション：文字列中の変数を変数値に置換する シングルクォーテーション：置換しない

マルチバイト概要

PHPレベル

UTF-8などのマルチバイト文字列を扱う関数としてmb_*が用意されている。

すべての文字列関数にmb_版があるわけではない。ない場合は「ご愁傷様」？？？

データベースレベル

MySQLに渡す文字列を確実にUTF-8にするには 設定をutf8ではなく、utf8mb4にすること！

ブラウザレベル

Content-Typeヘッダーの中で文字セットを指定する。

// PHP に対して、今後このスクリプトの中では UTF-8 文字列を使うことを伝える
mb_internal_encoding('UTF-8');

// PHP に対して、ブラウザに UTF-8 で出力することを伝える
mb_http_output('UTF-8');

preg_replace mb_ereg_replace の違い

preg_replace(array,array,string) がOK mb_ereg_replace(string,string,string) のみ

エンコーディング

mb_internal_encoding(“UTF-8”) 入力文字、出力文字、mbstringモジュールのエンコーディングを指定

機種依存文字

SHIFT_JISなどの文字コードでは①などは定義されていない。 Windows、Macそれぞれ別の割り当てをしているため、互換性がない。 UTF-8では定義されているので、OK

アポストロフィとシングルクォーテーション

‘:アポストロフィ ‘’:シングルクォーテーション(option+]で入力できる)

文字列の一致

strstr($url, ‘/shop/item’) $urlに’/shop/item’が入っていたら入っていたところから最後までの部分文字列を返す

stristrは大文字小文字を区別しない

strpos:文字列内の部分文字列が最初にあらわれる場所を見つける strpos(string $haystack , mixed $needle [, int $offset = 0 ] ) 文字列 haystack の中で、 needle が最初に現れる位置を探します。
