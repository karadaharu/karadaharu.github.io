vagrant上にrails環境を整備するまで

vagrant上にrails環境を整備するまで

$ vagrant init centos6
$ vi Vagrantfile
config.vm.synced_folder "./data", "/home/vagrant/linkum/"
$ mkdir data
$ vagrant ssh

$ sudo yum -y install git
$ sudo yum -y install gcc openssl openssl-devel libffi-devel readline-devel tcl-devel tk-devel gdbm-devel bison libxml2-devel patch gcc-c++ mysql-devel mysql-server　postgresql-devel file-libs file-devel
$ echo 'export PATH="$HOME/.rbenv/bin:$PATH"' >> ~/.bash_profile
$ echo 'eval "$(rbenv init -)"' >> ~/.bash_profile
$ exec $SHELL -l
$ rbenv --version
$ git clone https://github.com/sstephenson/ruby-build.git ~/.rbenv/plugins/ruby-build
$ rbenv install 2.0.0-p643

$ gem install rails // railsのバージョン指定しなくて大丈夫？
$ git clone https://github.com/Mister-22-Century/linkum.git
$ git checkout develop
$ git pull origin develop
$ bundle install

$ sudo vi /etc/my.conf
character-set-server = utf8
$ sudo /etc/rc.d/init.d/mysqld start
$ mysql -u root
> create user 'linkum'@'localhost' identified by 'linkum';
> grant all on *.* to 'linkum'@'localhost';
> create database linkum_development character set utf8;
$ vi config/database.yml
development:
  adapter: mysql2
  encoding: utf8
  reconnect: false
  database: [アプリ名]_development
  pool: 5
  username: development
  password: development
  host: localhost
  socket:
$ rake db:migrate

vagrant上のゲストOSのタイムゾーンがJSTじゃないと、 railsのファイル更新チェックがうまく機能しないので注意
