Rails入門

Rails入門

概要

バージョン 4.xを入れる

チュートリアル

小学生でもわかるRuby on Rails入門 で学ぶ。

ビューとコントローラーをつくる

rails g controller users index show

g:generate usersがコントローラー名 index, showがビュー名（いくつもつくれる？どうやってたす？）

scaffold:model view contriller 全部簡単にできる

RESTful

REST：対象となるリソースをURLを使って表し、 httpのGET、　POST、DELETE、PUTで処理を分ける

コントローラー

respond_to リクエストフォーマットに応じた処理をする

テンプレートエンジン

ERB:eRuby(テキストファイルにRubyスクリプトを埋め込む書式の仕様)のrubyによる実装 Haml:

eRuby <% … %> Rubyスクリプトをそのまま実行 <%= … %>　式を評価した結果を挿入

Haml

Gemfileに

gem 'haml-rails'
gem 'erb2haml'

で

bundle install
rake haml:replace_erbs

で全部Hamlになる。 railsの簡易サーバーを使っているときは一度再起動すればOK

Class

$ rails console
> Classname

フォームから入力した値を保存する

form_forでモデルを指定してフォームをつくる。 params[:modelname]でPOSTの値を取得できる。

データベースとしてMySQLを使う

CentOSに新しいMySQLをインストール http://d.hatena.ne.jp/sakaik/20140630/mysql_install_repository

RailsにMySQL http://qiita.com/itkrt2y/items/4ea5bfe90ca65e4a9ebd

rails new appname -d mysql

MySQLでdevelopment,test,production用のユーザーを合計3つ作成する

> create user uname1 idnetified by 'pass';
> grant all privileges on *.* to 'railsdev'@'localhost' identified by 'railspass';

MySQLはデフォルトでは謎のエンコーディングなので、設定を変えてからデータベースをつくること。 http://babiy3104.hateblo.jp/entry/2014/02/13/000219

rake db:create db:migrate

でとりあえずOK

CentOSでrails

簡易サーバーが立ち上がらない http://mymemo.weby117.com/develop/ruby-geminstall-error_3.html Javascripsエンジンが必要。 対処法としては,Gemfileに以下を追記する。 どうせ本当にサービス動かすときはnginxなど使うので、とりあえず。

gem 'therubyracer'

外から見えるようにIPアドレスを指定 rails server -b 192.168.33.13

MySQLの起動 sudo /etc/init.d/mysqld start

gemのインストール sudo gem mysql2

mysql.h is missing. please check your installation of mysql and try again. インクルードヘッダーが見つからない

yum install mysql-devel

-develは開発に必要なヘッダファイル等が含まれている。（ベータ版ではない！） そのアプリケーションを使って開発するときに必要。
