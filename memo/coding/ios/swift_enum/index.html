列挙型 enum

http://www.slideshare.net/tomohirokumagai54/enum-yhios-cocoa-kansai

ポイント

列挙型もオブジェクトなのでメソッドとか持てる

let b = Basket.Melon
let string = b.toString()

Raw値としてInt, Float, Stringが使える

enum Basket : String {
  case E = ""
  case Pineapple = "tar"
  case Melon = "hoge"
}
let str:String = Basket.Pineapple.rawValue

Associated Value:値を格納できる

enum Barcode {
  case UPCA(Int, Int)
  case QRCode(String)
}
let bar = Barcode.QRCode("hoge")
swich bar {
  case .QRCode(let prodCode):
    println(prodCode)
}

いちいちswitchを書くのはだるい→内部でメソッド化

enum Barcode {
  case UPCA(Int, Int)
  case QRCode(String)

  func qrcode() -> String? {
    switch self {
      case .QRCode(let prodCode):
        return prodCode
      default:
        return nil
    }
  }
}

インスタンス化するときに引数の意味が分かりづらい→明示的に名前をつける

enum Barcode {
  case UPCA(id:Int, val:Int)
  case QRCode(url:String)

  func qrcode() -> String? {
    switch self {
      case .QRCode(let prodCode:url):
        return prodCode
      default:
        return nil
    }
  }
}

列挙型で独自クラスを使うことができる

Row値の型に独自クラスを使える Row値はリテラルで指定

リテラル：数字の1や文字列”hello”などのプログラムの中に直接記述できる値のこと

enum Basket : MyClass {
  case Empty = 0.0
  case Apple = 1.0
  case Melon = 1.1
}

列挙型のRaw値を取得したときに

  1. 初めてインスタンスが生成される
  2. 変換イニシャライザが実行される
  3. 列挙子が対応するリテラルが渡される
