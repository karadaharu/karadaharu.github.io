ロケール：国際化・地域化 文字列、日付、通貨などの設定がよくなる

LC_Collate：文字列の並び順（漢字、濁音など） LC_CTYPE：文字の種類（大文字・小文字？）

その他LC_NUMERICなど。

表の結合

等価結合：両方満たすものだけ

select t1.* from table1 t1, table2 t2 where t1.code_target = t2.code

inner joinを使ってもできる。

select t1.* from table t1 inner join table2 t2 on t1.code_target = t2.code

集計

group byで集計できる

ダンプ

スキーマ：列の属性，制約，テーブル間のリレーションなどの定義

テーブルをコピー

あるテーブルのスキーマのみをSQL文で取り出す

pg_dump -U postgres -s -t table_name2 db_name > table_name2.sql

復元 psql dbname < table_name2.sql

バージョンアップ

pg_dumpとpostgresqlのバージョンが違った

パッケージではインストールしていないはず テーブルのダンプ pg_dumpとpostgresのバージョンが違うのでできない 選択肢 バージョンが違ってもできる方法を探す –ignore-versions だめ バージョンアップする

http://stackoverflow.com/questions/23793774/psql-and-pg-dump-version-mismatch?rq=1 複数の箇所にpg_dumpがインストールされているなら，シムリンクで対応 /usr/local/bin/pg_dump /bin, /usr/bin/にはない．たぶんない．

http://stackoverflow.com/questions/13997732/conflict-with-server-version-of-postgresql-9-2-1-and-pg-dump-version-9-1-4-rail 複数の箇所にインストールされている可能性がない場合は， どうやってインストールしたか確認

http://stackoverflow.com/questions/12836312/postgresql-9-2-pg-dump-version-mismatch?lq=1

結局， /usr/local/pgsql/bin/下のを使えばよかった．
