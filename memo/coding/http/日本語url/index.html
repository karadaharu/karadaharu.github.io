URLエンコード

URL/URIのファイル名やクエリ文字列などの一部としては使用できない記号や文字を、使用できる文字の特殊な組み合わせによって表記する変換規則。

日本語の場合、文字コードによってエンコードの結果は変わる

Macの濁音問題

Unicodeで「が」を表すとき、次のふたつの方法がある

  • 「が」（U+304C）の一文字で表す方法 すべての文字がこの方法で統一された形式を NFC（Normalization Form C。C は Composition＝合成 の意味）と呼ぶ

  • 「か」（U+304B）+「゙ 」（U+3099）の二文字で表す方法 すべての文字がこの方法で統一された形式を NFD（Normalization Form D。D は Decomposition＝分解 の意味）と呼ぶ

Macは後者を使うので、Macにある濁音を含むファイルをlinuxに送ると「か”」になる。

aリンクはUTF8-MACでつくられるのに、転送したときにファイルがUTF8になってしまうので、一致しなくなる。
