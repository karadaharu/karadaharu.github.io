pyenv

##pyenv Pythonのバージョンを管理する $ cd ~/Projects/sandbox/ $ pyenv local 3.3.3 $ pyenv version 3.3.3 (set by /Users/xxx/Projects/sandbox/.python-version) でディレクトリごとにバージョン指定できる

##virtualenv Pythonのライブラリのバージョンを管理する

http://qiita.com/maosanhioro/items/47a52f96fefba7126f9b

$ pyenv virtualenv 2.7.6 sandbox276
pyenv local sandbox276

$ pyenv uninstall 3.3.3
$ pyenv uninstall sandbox276
$ pyenv versions
system* 2.7.6 (set by /usr/local/opt/pyenv/version)

次はdjangoのインストール ついにpythonかける

プロジェクトごとに使うライブラリは結構変わったりして、たまに同じライブラリだけど違うバージョン使ったりするじゃん！ 問題は、pythonのimport文だけじゃバージョン指定できないんだよね。だから、先にパス通ってるものを勝手に採用されるんだけど、場合によってその挙動はネックだったりする。それを避けるためにvirtualenvはよく使われる。 簡単に言えばプロジェクトごとに使うライブラリを分けてくれる感じ！Python的にそれを使うことによって、プロジェクトAではライブラリXのバージョン1.2が入ってて、プロジェクトBでは同じライブラリXのバージョン1.6が入ってるから、importする時は確実使いたいバージョンが採用される。 そこはrubyのbundlerの方が圧倒的に優秀なんだけどなー いつかPython版を作ってみたいものだー
