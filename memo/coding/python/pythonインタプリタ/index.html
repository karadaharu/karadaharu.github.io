Pythonは環境がぐちゃぐちゃになりがち

インタプリタとは

コードを実行するとき，方法は大きく２つ． * インタプリタ方式 * コンパイラ方式

インタプリタ方式

コードを読み込んで，ただちに実行する．

コンパイラ方式

コンパイラはプログラムを実行しない．機械語に変換するだけ

Pythonのインタプリタ

OSXではPythonインタプリタはいくつかある． それぞれ，インストール先が違う． * デフォルト:/usr/bin/python * Pythonの公式サイトからインストーラをつかったもの:/usr/local/bin/python * pyenv:/Users/kazumasa/.pyenv/shims/python

→それぞれのインタプリタと同じところのpipを使ってあげないといけない →シンボリックリンク

今回の問題と対処方法

  • 問題:pyenv環境にpipでライブラリをインストールしたいのに，/usr/local/bin/pipが実行されてしまって，別のところにインストールされてしまう．
  • 対処:/usr/local/bin/pipを削除→which pipでpyenv内のpipが呼ばれていることを確認
