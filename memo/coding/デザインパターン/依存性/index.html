依存性

依存性

概要

制御の反転(Inversion of Control)

ライブラリはユーザーが呼び出して使う。フレームワークはフレームワークがメインになってユーザーのプログラムを制御する。これをIoCと呼ぶ。

Ralph Johnson氏とBrian Foote氏が書いた1988年の論文「Designing Reusable Classes」（http://www.laputan.org/drc/drc.html）で提唱された。

<?php
namespace Database;

class Database
{
    protected $adapter;

    public function __construct()
    {
        $this->adapter = new MySqlAdapter;
    }
}

class MysqlAdapter {}

これだとDatabaseクラスがアダプターと密接に結びついてしまう。

<?php
namespace Database;

class Database
{
    protected $adapter;

    public function __construct(MySqlAdapter $adapter)
    {
        $this->adapter = $adapter;
    }
}

class MysqlAdapter {}

こうすると依存関係を緩やかにできる。依存関係を外部から渡せるようになった。これでクラス自体の問題なのかそうでないのかが分かりやすくなるとかなのか？
