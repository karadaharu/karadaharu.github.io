bashを理解する

#bashを理解する .bash_profileと.bashrcの違い .bash_profileはログイン時のみ実行される .bashrcは新しくシェルが起動される度に実行される。.bash_profileから呼ばれる形でも実行される。

環境設定ファイルbash_profileの反映

$ source ~/.bashrc

パスを通す　~/.bashrcに下記をかいて，反映

export PATH=通したいディレクトリのパス:$PATH

$PATHはすでに通っているパス変数 先に書いてあるパスが優先されるので， $PATH:通したいディレクトリのパス，の順番だと同じコマンドの場合だめ

コマンドのエイリアスを作成（例：sublime textを起動するsublコマンド）

sudo ln -s "/Applications/Sublime Text 2.app/Contents/SharedSupport/bin/subl" /usr/local/bin/subl

シェルスクリプトで対話的作業を行う。

#!/usr/bin/expect
spawn ssh ebi@tempura.jp
expect "PASSWORD?"
send "password\n"
interact

ftpは定形フォーマットをftpコマンドに食わせてやればOK
