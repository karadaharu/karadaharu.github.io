electron tutorial

package.jsonの”main”に指定しているjavascriptを”メインプロセス”と呼ぶ

画面に対応するのがレンダープロセス BrowserWindowインスタンスをつくることでレンダープロセスがつくられる。

windowオブジェクト：ブラウザのウィンドウをさす。グローバルオブジェクトなので、window.documentはdocumentと書ける。←electronでも同じ

documentオブジェクト：ウェブページの部分。htmlのdomツリーとその他のプロパティ。document.headでがとれる。←electronでも同じ document.cookieはとれない。

クッキーにはhttponlyフラグがあって、これがたってると、jsからアクセスできない（セキュリティ）

https://github.com/atom/electron/issues/1142 クッキーというのはHTTP/HTTPSのコンテキストでしか意味をなさない。atom shellはファイルを読み込むので、クッキーは存在しない

対応

  1. localstrageとかに保存　これで
  2. https://github.com/hstove/electron-cookiesでdocument.cookieを偽装
  3. electronのsession.cookieを使う
  4. app.getPathのappDataのところにjsonとかで保存

表のjsで閉じるのを検知する(Electron)

window.onbeforeunload = function(e) {
    var remote = require('remote');
    var dialog = remote.require('dialog');
    var choice = dialog.showMessageBox(
            remote.getCurrentWindow(),
            {
                type: 'question',
                buttons: ['Yes', 'No'],
                title: 'Confirm',
                message: 'Are you sure you want to quit?'
            });
    return choice === 0;
};
